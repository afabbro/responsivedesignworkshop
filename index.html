    • How do we make mobile websites?

    • Design separate sites for mobile and desktop (sometimes a tablet version too)
    • Mobify it (3rd party)
    • Responsive design

    Pros and cons of designing separate sites

    Pros

    • Best performance
    • Highest level of detail control

    Pros and cons of designing separate sites

    Cons

    • Multiple websites to maintain
    • Poor redirection/no redirection for link sharing
    • Need to to managed multiple versions of content
    • Sharing a link via desktop on Twitter and a mobile users clicks on it - shitty experience

    Pros and Cons of Mobify (3rd party)

    Pros:

    • Can use it on any existing desktop site, with existing security
    • Serves up the page to mobile users at the same URL
    • Separate templates mean conditonal content loading
    • Transform DOM elements into other DOM elements

    Pros and Cons of Mobify (3rd party)

    Cons:

    • AJAX (content loaded asynchronously by JS) - based sites end up being slower
    • Relies on 3rd party Javascript (code could break on upgrade)
    • Learning curve - need to be familiar with Javascript, command line tools etc.

    Pros and Cons of Responsive Design

    Pros:

    • You only make one website!
    • Not just about the device - resize the browser and it still looks good!

    Pros and Cons of Responsive Design

    Cons:

    • Development time
    • Worse performance (if you don't know what you're doing)
    • We're giving you the barebones, but to do a richly interactive site requires a lot of work
    • All the magic to resize and refit the websites (including extra helper scripts) makes website slower - unless you REALLY know what you're doing

    Designing for Mobile

    You will get the best results if you design for mobile first.

    • Don't design to a fixed grid and then try to make it fluid later. Fluid grids/layout all the way!
    • Assume that you're designing for touch screens (iOS HIG suggests 44px^2 as tap target)
    • Large headings work better on wider layouts
    • Angelina tried to do this and it's the wrong way to do it
    • We don't have a truly reliable way to test for touch interaces, though there are libraries (Modernizr) that do a good job, better just to design as though it's a touch interface.
    • Large headings on small screens means unecessary scrolling

    Designing for Mobile

    You will get the best results if you design for mobile first.

    • Navigation is less important than you think sometimes
    • Get rid of superfluous content
    • Don't be afraid of whitespace
    • If you're a business: make it easy to find you
    • Content-first, navigation second for mobile experiences. Don't keep people from what they want. Ever been trying to
    • ...But have a link to your desktop site somewhere if you're trimming a lot
    • Don't cramp up your design
    • Many websites direct you to desktop for contact info. Separate contact page is okay if you don't want it taking up space as a footer, but make the way to get there clear as the blinding sun.

function linkify( selector ) {
  if( supports3DTransforms ) {
    
    var nodes = document.querySelectorAll( selector );

    for( var i = 0, len = nodes.length; i < len; i++ ) {
      var node = nodes[i];

      if( !node.className ) ) {
        node.className += ' roll';
      }
    };
  }
}
